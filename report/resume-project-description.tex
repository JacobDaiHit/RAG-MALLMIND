\datedsubsection{\textbf{MallMind：基于 RAG 的多模态电商智能导购系统}}
{\textit{2026.5 -- 至今}}
\begin{itemize}
  \item 面向电商导购与 PC 装机场景，参与设计并实现基于
    \textcolor{HITBlueAux2}{FastAPI + RAG + Milvus + LLM}
    的后端 Agent 链路，覆盖意图路由、需求解析、商品召回、结构化过滤、
    可解释排序、事实校验和 SSE 流式输出，共 8 个工具处理器；

  \item 设计 \textcolor{HITBlueAux2}{LLM 优先 + 本地规则降级}双通道路由：
    LLM 经结构化 Prompt（8 工具 JSON Schema + 品类枚举 + few-shot）
    输出 tool\_call，本地 14 步决策树独立判定，guard 逻辑做争议仲裁
    （LLM 推荐但含闲聊信号 → 回退本地结果），减少低置信度越界路由；

  \item 完成商品证据切片与向量入库，将电商与 PC 配件数据切分为
    profile/SKU/FAQ/review 四类 chunks，接入 DashScope
    \texttt{text-embedding-v4}（1024 维）+ BM25 稀疏索引（持久化 + 增量更新），
    实际完成 \textcolor{HITBlueAux2}{884 条证据切片}入库（400 电商 + 484 PC）；

  \item 构建三层检索管道：12 步确定性过滤（品类/库存/品牌白名单/排除词/预算等，
    每步空结果安全降级）→ 向量 + BM25 混合检索经 RRF 融合 →
    7 维可解释评分重排（场景/属性/价格/口碑/库存/SKU/证据命中）+
    20 种动态权重调整 + MMR 多样性控制；

  \item 实现\textcolor{HITBlueAux2}{事实校验层}防 LLM 幻觉：
    product\_id 存在性验证、价格偏差 >30\% 自动修正为真实价格、
    库存标注，失败率 >50\% 触发降级，确保商品信息均来自真实库而非 LLM 生成；

  \item 实现 PC 装机子模块，围绕 8 类配件做候选剪枝、预算约束和兼容性校验
    （socket/内存代数/电源功率/机箱尺寸），结合 session 历史支持多轮方案修改；
    购物车 CRUD 采用计划 + 确认两步模式（60s TTL），通过三级 ID 解析和
    模糊匹配支持"删除第二个""把华为数量改成 5"等指代操作；

  \item 重构多轮会话管理，抽象 InMemorySessionStore 与 RedisSessionStore，
    保存消息/购物车/装机历史/topic memory/trace；配合查询改写器实现代词消解和
    属性继承，三层话题切换检测（显式信号 → 品类感知 → PC 豁免）防止上下文漂移；
    补充向量库健康检查、召回评估和消融评估脚本，用 hit@k、MRR、constraint
    violation 等指标迭代边界逻辑，21 cases / 136 turns 全量回归路由成功率 100\%。
\end{itemize}
